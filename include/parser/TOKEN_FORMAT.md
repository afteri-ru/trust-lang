# TOKEN_FORMAT.md — Формат токенов парсера

## Описание

Файл описывает формат и правила создания токенов лексером (Flex).

## Функции создания токенов

### `Lexer::token(e, kind, len, skip_start, skip_end)`

Создаёт токен из текущей позиции в буфере.

| Параметр     | Тип        | Описание                              | По умолчанию |
|-------------|------------|--------------------------------------|:------------:|
| `e`         | `Lexer*`   | Указатель на лексер                  |              |
| `kind`      | `int`      | Вид токена (из `ParserToken::Kind`)  |              |
| `len`       | `unsigned` | Длина всей лексемы                   |              |
| `skip_start`| `unsigned` | Смещение от **начала** лексемы        | `0`          |
| `skip_end`  | `unsigned` | Смещение от **конца** лексемы         | `0`          |

**Примеры:**
```lex
TK(DOC_AFTER)           — полная длина, без обрезки
TK(DOC_BEFORE, 3, 2)   — пропустить 3 символа от начала, 2 от конца
TK(NUMBER)
```

### `Lexer::token_stateful(e, kind, open_len, close_len)`

Создаёт токен для stateful-состояний. Контент вычисляется между `content_start` и текущим `offset`.

| Параметр     | Тип        | Описание                               | По умолчанию |
|-------------|------------|---------------------------------------|:------------:|
| `e`         | `Lexer*`   | Указатель на лексер                   |              |
| `kind`      | `int`      | Вид токена (из `ParserToken::Kind`)   |              |
| `open_len`  | `unsigned` | Длина открывающего делимитера         |              |
| `close_len` | `unsigned` | Длина закрывающего делимитера         |              |

**Примеры:**
```lex
TK_STATEFUL(DOC_BEFORE, 3, 2)
TK_STATEFUL(EMBED, 2, 2)
TK_STATEFUL(EVAL, 1, 1)
```

## Макросы

| Макрос            | Описание                                    |
|-------------------|--------------------------------------------|
| `TK(kind, ...)`   | Обычный токен, `...` → `skip_start, skip_end` |
| `TK_STATEFUL(...)`| Токен из stateful-состояния                |
| `ADV(n)`          | Продвинуть offset на `n`                   |

## Stateful-состояния

| Состояние              | Открытие      | Закрытие   | open_len | close_len |
|-----------------------|:-------------:|:----------:|:--------:|:---------:|
| `state_DOC_BEFORE`    | `/**`         | `*/`       | 3        | 2         |
| `state_EMBED`         | `{%`          | `%}`       | 2        | 2         |
| `state_MACRO_STR`     | `@@@`         | `@@@`      | 3        | 3         |
| `state_STRWIDE`       | `"`           | `"`        | 1        | 1         |
| `state_STRCHAR`       | `'`           | `'`        | 1        | 1         |
| `state_STRWIDE_RAW`   | `r"`          | `"`        | 2        | 1         |
| `state_STRCHAR_RAW`   | `r'`          | `'`        | 2        | 1         |
| `state_EVAL`          | `` ` ``      | `` ` ``   | 1        | 1         |
| `state_ATTRIBUTE`     | `@[`          | `]@`       | 2        | 2         |

## Правила именования

- Все токены записываются прописными буквами с подчёркиванием: `EMBED`, `MACRO_STR`.
- Токены начинаются с `TK_` или `%token` в зависимости от контекста.
- Категории: `FLEX_LEXEME`, `BISON_TOKEN`, `Expr`, `Stmt`, `Decl`, `Root`.
