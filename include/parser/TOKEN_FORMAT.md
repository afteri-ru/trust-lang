# tokens.def — Формат описания токенов

## Синтаксис

```
TOKEN(NAME, VALUE, FLAGS)
```

Каждая лексема описывается строго на **одной строке**. Поддерживаются только `//` комментарии.

## Поля

| Поле | Описание | Значения |
|------|----------|----------|
| `NAME` | Имя токена | Должно соответствовать конвенции именования (см. ниже) |
| `VALUE` | Числовое значение токена | `0` — только для END (EOF)<br>`AUTO` — автоматическое значение (фиксируется в `tokens.lock`)<br>`N` — явное число (проверяется на конфликты) |
| `FLAGS` | Битовая маска классификации | Один или несколько флагов через `\|` |

## Флаги (TokenFlag)

| Флаг | Значение | Описание |
|------|----------|----------|
| `FLEX_LEXEME` | 1 | Лексема возвращается Flex lexer'ом |
| `BISON_TOKEN` | 2 | Токен участвует в грамматике Bison (как `%token` или `%nterm`) |
| `Expr` | 4 | Узел выражения в AST |
| `Stmt` | 8 | Узел оператора/блока в AST |
| `Decl` | 16 | Узел объявления в AST |
| `Root` | 32 | Корневой узел AST |

### Правила совместимости флагов

- `FLEX_LEXEME` + AST-флаг (`Expr`/`Stmt`/`Decl`/`Root`) — **запрещено**
- `BISON_TOKEN` + несколько AST-флагов — **запрещено** (ровно один или ноль)
- Каждый токен должен иметь хотя бы один флаг
- `FLEX_LEXEME` без AST-флага — классифицируется как терминал
- `BISON_TOKEN` + AST-флаг — классифицируется как узел грамматики

## Конвенция именования

| Стиль имени | Обязательные флаги | Назначение |
|-------------|-------------------|------------|
| `UPPER_CASE` | `FLEX_LEXEME` | Flex-лексемы (терминалы) |
| `CamelCase` | AST-флаг (`Expr`/`Stmt`/`Decl`/`Root`) | Bison-нетерминалы и/или AST-узлы |
| `lower_case` | Только `BISON_TOKEN` (без FLEX_LEXEME и AST) | Зарезервировано для будущих Bison-нетерминалов (пока не используется) |

## Ограничения

1. **END** должен иметь значение `0` (стандарт Bison/yacc)
2. **Ровно один** токен с флагом `Root` ( Program )
3. Не может быть токенов без флагов
4. Значения AUTO начинаются с 256 и не должны конфликтовать с явными значениями
5. Значения фиксируются в `tokens.lock` — при изменении вручную файл нужно обновить
